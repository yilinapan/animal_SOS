# 子報告：寵物走失協尋 + 社群動物求助文生態（含競品檢查）

研究日期：2026-06-12。方法說明：與其他子報告相同，多數網站 WebFetch 回傳 403，內容取自 WebSearch 摘要。**特別重要的研究限制**：Threads、Dcard、FB 社團、PTT 多為封閉或弱索引平台，外部搜尋引擎幾乎抓不到貼文原文與回覆串，故「社群求助文的頻率、回覆品質」無法量化驗證，只能以間接證據與推論呈現（見第四節）。標籤：**[事實]**／**[軼事]**／**[推論]**。

## 一、走失協尋管道現況

- **[事實]** 法定流程：寵物走失須於 5 日內至「寵物登記管理資訊網」(pet.gov.tw) 辦理遺失申報，資訊公告於該網供全國查詢；收容所/獸醫院掃晶片即可聯絡飼主。[來源: https://www.pet.gov.tw/Web/O205.aspx ；https://www.tcapo.gov.taipei/News_Content.aspx?n=10457D24D71EC09E&s=78FA051A62FBB4B8]
- **[事實]** 晶片制度的實效缺口：報導指近 3 成寵物晶片「沒有飼主資料」（打了晶片但未完成登記），登記資料錯誤/過期也直接影響找回。[來源: https://health.ettoday.net/news/1689819]
- **[事實]** 晶片有用但覆蓋不足的佐證：2018 年收容所收到 11,789 隻走失動物，僅 2,611 隻犬因有晶片得以快速通知飼主領回。[來源: 搜尋摘要，出處疑為 https://www.taeanimal.org.tw/model_topics.php?sn=1569 或同批衛教文，未核原文，引用前請確認]
- **[事實]** 2026 年起家貓納入強制寵物登記。[來源: https://edh.tw/lohas/article/31885]
- **[事實]** 民間管道生態：FB 走失協尋社團（如「狗狗走失協尋互助 Missing Dogs Taiwan」「寵物認養/領養/送養/協尋/救援/愛心訊息交流專區」、粉專「寵物走失協尋專區」）、PTT dog/cat/pet 及地區板、各式協尋部落格整理文（曾列出 20 個網路協尋專區）、新興媒合平台「浪途通報」(noramichi.app) 涵蓋救援/送養/認養/協尋。[來源: https://www.facebook.com/groups/121649277854432/ ；https://www.facebook.com/dogcat.love.home/ ；https://danapai.pixnet.net/blog/post/24200419 ；https://noramichi.app/]

## 二、走失後黃金時間與飼主常見錯誤

- **[事實]** 衛教文強調走失後「黃金 3 小時」內立即行動最關鍵（就近搜索、貼公告、通報）。[來源: https://blog.dogtorstray.com/lost/（浪犬博士）；數字據搜尋摘要]
- **[事實]** 常見錯誤一：登記資料錯誤——有來源稱逾 35% 協尋失敗與登記資料不正確有關（飼主簽名時未核對系統上的聯絡資料）。[來源: 搜尋摘要（出處待核，疑為 lunaexplorer.tw/chapter/718 等衛教文），數字未經原文核對]
- **[事實]** 常見錯誤二：只在社群發文、未做法定遺失申報，收容所端因此無法比對。[推論，依據 pet.gov.tw 流程與衛教文]
- **[事實]** 北市動保處整理「寵物走失四種協尋方法」（遺失申報、收容所查詢、社群發布、張貼啟事）。[來源: https://www.tcapo.gov.taipei/News_Content.aspx?n=10457D24D71EC09E&s=78FA051A62FBB4B8]
- **[推論]** 痛點本質：管道破碎（官方系統＋數十個 FB 社團＋PTT＋協尋平台互不相通），飼主在慌亂中要同時操作多平台，黃金時間被流程吃掉。

## 三、社群求助文生態（Threads/FB/Dcard/PTT）

- **[事實]** Dcard 有「撿到鳥」專屬話題頁（聚合多篇貼文），並有「撿到鸚鵡多久可以收編？」等貼文——顯示「撿到動物」在 Dcard 是反覆出現的固定案型。[來源: https://www.dcard.tw/topics/撿到鳥 ；https://www.dcard.tw/f/pet/p/255046258]
- **[事實]** PTT 生態：dog/cat/Aves/Pet_Get 等板長期有撿到動物、求助、送養文；cat 板甚至需訂板規處理放養/餵養爭議文。[來源: https://www.ptt.cc/bbs/Pet_Get/index.html ；https://www.ptt.cc/bbs/cat/M.1712111628.A.9E7.html ；https://www.ptt.cc/bbs/Aves/M.1395862224.A.5F4.html]
- **[事實]** 常見案型（由衛教文反覆回應的主題反推）：撿到雛鳥/幼鳥、路上受傷貓狗、撿到奶貓、開車撞到貓狗、發現屍體、走失協尋。每一類都有多篇民間 SOP 文存在，正因同樣問題在社群重複出現。[來源: https://e-info.org.tw/node/37240 ；https://litomon.com/blog/takecarekitten/ ；https://www.vet.com.tw/news/knowledge-car-accident-0126 ；https://www.kingautos.net/317305]
- **[事實]** 錯誤建議氾濫的佐證（間接）：救傷單位與衛教文反覆糾正的迷思——幼鳥不需要撿（「愛心變綁架」）、不能餵牛奶/麵包、撿到鸚鵡想直接收編（實為遺失物，依法應送交）等，顯示社群上第一時間的「熱心回覆」常是錯的。[來源: https://www.tcapo.gov.taipei/News_Content.aspx?n=67993BA7C146BC76&s=084539FF69CF464F ；https://pansci.asia/archives/164058 ；https://www.dcard.tw/f/pet/p/255046258]
- **[事實]** Threads 上確實存在動物求助/救援類貼文：例如「如果遇到受傷的流浪動物可以打 1999，政府會請你送合作動物醫院」的衛教貼文、花蓮光復災後動物救援的現場貼文、「捐款給浪浪到底要給哪個機構」的求助文。[來源: https://www.threads.com/@im.rron/post/C9uHseVTjud ；https://www.threads.com/@joliechi/post/DPK8FPqk77R ；https://www.threads.com/@6rey_____/post/DUS4eRqk5Dk]
- **[事實]** Threads 同時是詐騙溫床：假「免費領養」貼文誘加 LINE 騙個資/卡號，MyGoPen 已發查核警示——求助者在社群可能遇到的不只是錯誤建議，還有詐騙。[來源: https://www.mygopen.com/2026/01/threads-fraud.html]
- **[誠實回報／研究限制]** Threads 內容對外部搜尋引擎封閉性高：本次無法系統性檢索「Threads 撿到鳥」等求助原文與其回覆串，無法估計求助文頻率、回覆數量與品質分布。上列 Threads 連結是搜尋引擎偶然收錄的個別貼文。若要做嚴謹的量化生態研究，需以 Threads 帳號實際登入觀察/抓取，建議列為下一步（在站內搜尋「撿到」「受傷的貓」「幼鳥」等關鍵字並抽樣記錄回覆品質）。

## 四、競品檢查：有沒有人系統性在回覆這類貼文？

- **[事實]** 機構帳號存在但非「回覆型」：TSPCA 有官方 Threads 帳號 (@twspca)，主要做倡議與案件宣傳。[來源: https://www.threads.com/@twspca]
- **[事實]** 浪途通報 (noramichi.app)：民間「通報媒合平台」，涵蓋救援/送養/認養/遺失/協尋，提供制式欄位與案件管理——是目前最接近「結構化處理民眾通報」的競品，但它要求使用者離開社群到其平台填單，而非到貼文底下回覆。[來源: https://noramichi.app/]
- **[事實]** 台灣動物緊急救援小組 (savedogs.org)：只受理重傷重症流浪犬貓救援，明示不收容棄養、不協尋走失。[來源: https://www.savedogs.org/RescueReport]
- **[推論／結論]** 在可取得的證據範圍內，**未發現任何帳號（人工或 AI）系統性地在 Threads/Dcard/PTT 的「撿到動物/動物求助」貼文下即時回覆正確 SOP**。現有玩家不是「平台型」（要人主動來填單）就是「機構型」（發自己的內容）。回覆端目前由隨機網友承擔，品質參差且混有詐騙——這正是 Animal SOS 的空白市場。此結論受限於上述 Threads 封閉性，建議登入站內二次驗證。

## 五、對 Animal SOS 的意涵

1. **競品空窗**：沒有 reply-bot 型競品；Animal SOS「主動到貼文下回覆」的形態在現有生態中是獨特卡位。[推論]
2. **高頻案型可預先模板化**：撿到幼鳥（春夏尖峰）、奶貓、路傷貓狗、撞到動物、走失協尋——五大案型涵蓋多數求助文，各自已有權威 SOP 可引用（北鳥、北市動保處、獸醫院衛教）。[推論]
3. **反詐騙是信任切入點**：在求助文下提醒「免費領養加 LINE 多為詐騙」可建立帳號公信力。[推論]
4. **走失案型的回覆公式**：黃金 3 小時行動清單＋「5 日內 pet.gov.tw 遺失申報」＋該縣市收容所查詢連結＋主要 FB 社團清單——一則回覆整合官方與民間破碎管道。[推論]
5. **資料缺口即機會**：本研究無法量化 Threads 求助文頻率；建議產品啟動前先以 Threads 站內關鍵字監測 2–4 週，建立案型/頻率基線，同時直接形成回覆目標清單。[推論]

## 七、研究限制

1. **Threads 封閉性**：無法從外部系統性檢索貼文與回覆串，求助文「頻率/回覆品質」皆為質性推論，未量化。Dcard/FB 社團/PTT 內文同樣多數不可達。
2. 「近 3 成晶片無飼主資料」「35% 協尋失敗因登記資料錯誤」「黃金 3 小時」「2018 年 11,789/2,611」等數字皆來自搜尋摘要，未核對原文，引用前務必開原連結確認。
3. 競品檢查僅及搜尋引擎可見範圍；可能存在未被收錄的 Threads/LINE 社群內的系統性回覆者。
